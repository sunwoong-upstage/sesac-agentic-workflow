## 3. 전략적 서비스 도출

### 3.1 성공 사례 분석

**사례 1: Artifact (2024)** [예시]
- 도입 기술: LLM 기반 개인화 추천 + 요약
- 성과: 사용자 참여도 향상 [구체적 수치 확인 필요]
- 성공 요인: 직관적 UX, AI 요약 품질

**사례 2: 네이버 AI 뉴스 요약** [예시]
- 도입 기술: 자체 LLM 기반 요약
- 성과: 뉴스 소비 효율성 향상
- 성공 요인: 방대한 사용자 데이터 활용

### 3.2 실패 사례 분석: 일부 AI 뉴스봇

**실패 원인 분류**:
- 기술적: 단순 FAQ 수준, 맥락 이해 부족
- 데이터: 실시간 정보 연동 미흡
- UX: 대화형이 아닌 일방적 정보 제공

**교훈**: 단순 정보 제공이 아닌 대화형 상호작용 필수, 실시간 정보 연동 중요

### 3.3 차별화 전략

1. **Plan-and-Solve**: 계획 → 조사 → 합성 단계별 분해 (LangGraph 워크플로우)
2. **기본 메모리 구성**: 대화 체크포인트(MemorySaver, LangGraph 필수) + 사용자 프로필 저장(InMemoryStore, 관심사 누적)
3. **Evaluator-Optimizer Loop**: 자동 품질 검증 및 재생성
4. **RAG 기반**: Hallucination 방지, 사실 기반 응답 (FAISS + 뉴스 아카이브)
5. **멀티 도구**: RAG + 날짜 계산 + 실시간 웹 검색 조합

## 4. 서비스 상세 기획

### 4.1 핵심 가치

**서비스명**: News Chatbot Agent

**정의**: AI가 사용자의 뉴스 관심사를 학습하여 관련 뉴스를 검색, 요약, 제공하는 대화형 비서

| 가치 | 제공 내용 |
|------|----------|
| 시간 절약 | 수십 개 기사 → 핵심 요약 제공 |
| 맥락 이해 | 날짜/시간 기반 뉴스 필터링 |
| 개인화 | 관심 주제 학습 맞춤 추천 |
| 신뢰성 | 출처 명시, 사실 기반 응답 |

### 4.2 타겟 유저 정의

**Primary: 바쁜 직장인**

| 항목 | 내용 |
|------|------|
| 기본 정보 | 김뉴스 (30세, IT 기업 재직, 서울 거주) |
| 상황/니즈 | 출근 전 10분에 핵심 뉴스 파악 필요 |
| 행동 패턴 | 포털 뉴스 훑어보기, 관심 분야만 클릭 |
| Pain Points | 시간 부족, 중요 뉴스 놓침, 맥락 파악 어려움 |

**Secondary: 투자자/리서처**

| 항목 | 내용 |
|------|------|
| 기본 정보 | 이투자 (35세, 금융권 종사, 강남 거주) |
| 상황/니즈 | 특정 기업/산업 뉴스 모니터링 필요 |
| 행동 패턴 | 키워드 검색, 뉴스 알림 설정, 히스토리 추적 |
| Pain Points | 분산된 정보, 과거 뉴스 검색 어려움 |

### 4.3 가치 제안 (Value Proposition)

**서비스 본질**
"AI 뉴스 비서가 관심사에 맞는 뉴스를 찾아 핵심만 전달합니다"

**사용자 핵심 가치 3가지**

| 가치 | 기존 방식 | AI 에이전트 | 개선 |
|------|----------|-------------|------|
| 뉴스 검색 | 직접 검색/스크롤 | 대화로 요청 | **시간 80% 절약** |
| 맥락 이해 | 여러 기사 비교 | 통합 요약 제공 | **이해도 향상** |
| 개인화 | 수동 알림 설정 | 자동 관심사 학습 | **만족도 향상** |

### 4.4 사용자 여정 시나리오

**1단계: 시작 (문제 인식)**
- 김뉴스: 아침 출근 전 엔비디아 관련 뉴스 확인 필요
- 포털 뉴스는 너무 많고, 어떤 게 중요한지 모름
- Pain Point: 정보 과부하, 시간 부족

**2단계: 발견 (서비스 접근)**
- "AI 뉴스 챗봇" 검색 → News Chatbot Agent 발견
- 대화형으로 뉴스 요청 가능

**3단계: 사용 (핵심 기능 경험)**
```
사용자: "최근 일주일간 엔비디아 관련 뉴스 알려줘"

[Agent 프로세스]
1. 의도 분류: news_search
2. 날짜 계산: 7일 전 ~ 오늘
3. Research:
   - RAG 검색: "엔비디아"
   - 웹 검색: "엔비디아 뉴스 최신"
4. Synthesize: 주요 뉴스 3개 요약

Agent 응답:
📰 엔비디아 최근 1주일 뉴스 요약

1. **CES 2025 신제품 발표** (1월 27일)
   - 차세대 GPU 아키텍처 공개
   - 데이터센터용 신제품 라인업

2. **분기 실적 발표** (1월 25일)
   - 매출 전년비 200% 성장
   - AI 칩 수요 폭발

3. **삼성전자 협력 발표** (1월 24일)
   - HBM 공급 계약 체결
   - 파운드리 협력 확대

💡 관련 키워드: AI, 반도체, GPU, 데이터센터
```

**4단계: 해결 (목표 달성)**
- 5분 만에 핵심 뉴스 3개 파악
- 맥락과 함께 이해
- 관련 키워드로 추가 탐색 가능

**5단계: 만족 (가치 실현)**
- "매일 아침 이렇게 확인하면 좋겠다"
- 사용자 프로필에 관심 주제 저장 (엔비디아, AI, 반도체)

### 4.5 핵심 기능 정의

**기능 1: 뉴스 아카이브 검색 (search_news_archive)** [핵심]

| 항목 | 내용 |
|------|------|
| 목적 | 저장된 뉴스 아카이브에서 관련 기사 검색 |
| Input | 사용자 질문 (예: "삼성전자 실적 뉴스") |
| AI 처리 | 벡터 DB 검색 → 관련 문서 추출 (상위 3개) → LLM 답변 생성 |
| Output | 관련 뉴스 기사 (카테고리, 제목, 날짜, 본문 포함) |
| 성공 기준 | 정보 정확도 90% 이상, 응답 시간 3초 이내 |
| 구현 방안 | FAISS 벡터 DB + Upstage Embeddings |

**기능 2: 날짜 범위 계산 (calculate_date_range)** [핵심]

| 항목 | 내용 |
|------|------|
| 목적 | 상대적 날짜 표현을 실제 날짜로 변환 |
| Input | 시간 값 (time_value), 단위 (time_unit: days/weeks/months) |
| AI 처리 | 규칙 기반 계산 (현재 날짜 - delta) |
| Output | 시작일, 종료일 (YYYY-MM-DD 형식) |
| 성공 기준 | 정확한 날짜 계산 |
| 구현 방안 | Python datetime + timedelta |

**기능 3: 실시간 뉴스 검색 (search_recent_news)** [핵심]

| 항목 | 내용 |
|------|------|
| 목적 | 최신 뉴스 실시간 검색 (Serper API) |
| Input | 검색 쿼리 (예: "삼성전자 최신 뉴스") |
| AI 처리 | 웹 검색 API 연동 → 결과 요약 |
| Output | 뉴스 검색 결과 (상위 5개) |
| 성공 기준 | 정보 최신성 (24시간 이내), 응답 시간 10초 이내 |
| 구현 방안 | Serper API (Google 뉴스 검색) |

### 4.6 기술 아키텍처

**시스템 구성**
```
[사용자 입력] → [의도 분류 노드] → [에이전트 워크플로우]
  ├─ 선호도 추출 노드: 주제/키워드/날짜 범위 추출
  ├─ 계획 노드: 요청 분해 및 실행 계획 수립
  ├─ 조사 노드: RAG 검색 + 날짜 계산 + 웹 검색 병렬 수행
  ├─ 합성 노드: 최종 응답 생성
  └─ 평가 노드: 품질 검증 (필요시 재생성, 최대 2회)
     ↓
[메모리: 체크포인트(MemorySaver) + 프로필(InMemoryStore)] → [메모리 저장 노드]
     ↓
[응답 출력]
```

**기술 스택**

| 레이어 | 기술 | 역할 |
|--------|------|------|
| LLM | Upstage Solar Pro | 자연어 이해·생성 |
| Orchestration | LangGraph | 워크플로우 관리 |
| RAG | FAISS + Embeddings | 뉴스 아카이브 검색 |
| Memory | InMemoryStore | 사용자 프로필 저장 |
| Tools | Custom + Serper | 날짜 계산·웹 검색 |
| State | TypedDict | 워크플로우 상태 |

## 결론

**3가지 핵심 강점**:

1. **Plan-and-Solve**: 체계적 분해로 정확한 뉴스 검색
2. **기본 메모리 활용**: 대화 맥락 유지 + 사용자 관심사 기록
3. **3가지 도구 통합**: RAG + 날짜 계산 + 웹 검색

**목표**: 뉴스 소비의 효율화, 정보 과부하 해소, 개인화된 뉴스 경험 제공
